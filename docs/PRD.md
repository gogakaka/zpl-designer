## 1. 개요 (Overview)

### 1.1 제품 비전
라벨 프린터용 ZPL(Zebra Programming Language) 코드를 손으로 작성하는 일은 좌표를
dot 단위로 계산하고 수십 개의 `^` 명령어를 외워야 하는 고통스러운 작업이다.
**ZPL Designer**는 사용자가 브라우저에서 라벨을 시각적으로(WYSIWYG) 그리기만 하면
정확한 ZPL II 코드를 자동으로 생성해 주는 디자이너 툴이다.

### 1.2 문제 정의
- ZPL은 텍스트 기반 명령어 언어로, 비개발자가 직접 다루기 어렵다.
- 요소의 위치/크기를 dot 단위로 일일이 계산해야 하며, DPI(203/300/600)가 바뀌면
  모든 좌표를 다시 환산해야 한다.
- 기존 상용 툴(ZebraDesigner, BarTender, NiceLabel)은 설치형이거나 유료이며
  학습 곡선이 가파르다.
- 디자인 결과를 즉시 ZPL 코드로 확인·복사·재사용하기 어렵다.

### 1.3 제품 목표
1. 드래그 앤 드롭만으로 라벨을 디자인하고 즉시 ZPL II 코드를 추출한다.
2. 화면에 보이는 결과와 실제 프린터 출력이 일치하도록 dot 정확도 렌더링을 제공한다.
3. 설치 없이 브라우저에서 동작하며, 모든 데이터는 로컬에 안전하게 저장된다.
4. ZPL로 표현 가능한 **모든 도형·요소**(텍스트, 1D/2D 바코드, 선, 박스, 원, 타원,
   대각선, 이미지, 테이블)를 지원한다.

### 1.4 핵심 가치 제안
- **쉬움**: ZPL 문법을 몰라도 라벨을 만든다.
- **정확함**: 캔버스 좌표가 곧 ZPL dot 좌표 — 출력 오차 최소화.
- **즉시성**: 디자인하는 즉시 ZPL 코드가 실시간 갱신된다.
- **무설치·로컬**: 브라우저만 있으면 되고, 데이터는 사용자 기기에만 저장된다.

### 1.5 한 줄 정의
> "브라우저에서 라벨을 그려 ZPL II 코드를 추출하는 WYSIWYG 라벨 디자이너."

## 2. 대상 사용자 & 유스케이스

### 2.1 페르소나
| 페르소나 | 설명 | 핵심 니즈 |
| --- | --- | --- |
| **현업 운영자** | 물류·창고·소매 매장에서 라벨을 제작·인쇄 | ZPL 지식 없이 빠르게 라벨 디자인 |
| **개발자** | ERP/WMS/POS에 라벨 출력 기능을 연동 | 검증된 ZPL 템플릿 + 변수 바인딩 코드 |
| **라벨 디자이너** | 가격표·태그·배송 라벨 양식을 설계 | 정밀한 정렬·치수 제어, 템플릿 재사용 |
| **QA/지원 담당자** | 기존 ZPL 코드를 검토·수정 | ZPL 가져오기 → 시각적 확인 → 수정 |

### 2.2 핵심 유스케이스
1. **배송 라벨 제작**: 4"×6" 라벨에 주소 텍스트 + Code128 바코드 + 로고 이미지 배치 → ZPL 추출.
2. **가격표 제작**: 소형 라벨에 상품명·가격 텍스트 + EAN-13 바코드 배치.
3. **자산 태그 제작**: QR 코드 + 자산번호 텍스트 + 테두리 박스.
4. **양식 템플릿화**: 변수 필드(`{{상품명}}`)를 포함한 ZPL 템플릿을 만들어 시스템에 연동.
5. **데이터 일괄 적용**: CSV를 불러와 여러 레코드를 같은 양식에 머지해 미리보기.
6. **기존 ZPL 수정**: 외부에서 받은 ZPL 코드를 붙여넣어 캔버스에서 시각적으로 편집.

## 3. 용어 정의 & ZPL 기초

### 3.1 용어집
| 용어 | 의미 |
| --- | --- |
| ZPL / ZPL II | Zebra 라벨 프린터 명령어 언어. 본 제품의 출력 포맷. |
| dot | ZPL의 최소 단위. 모든 좌표·크기는 dot으로 표현. |
| DPI / dpmm | 인치당/밀리미터당 dot 수. 203dpi=8dpmm, 300dpi=12dpmm, 600dpi=24dpmm. |
| 요소(Element) | 캔버스에 배치되는 디자인 단위(텍스트·바코드·도형 등). |
| 라벨(Label) | 하나의 인쇄 매체 면. 캔버스 = 라벨 1장. |
| 프린터 프로필 | DPI·인쇄 폭·폰트셋 등 출력 대상 정보의 묶음. |
| HRT | Human Readable Text. 바코드 하단에 표시되는 사람이 읽는 숫자/문자. |

### 3.2 ZPL II 개요
- ZPL 라벨은 `^XA`(시작) … `^XZ`(끝)로 감싼다.
- 각 필드는 `위치(^FO) → 속성(^A/^BY/^GB 등) → 데이터(^FD) → 종료(^FS)` 순서.
- 설정 명령(`^PW`, `^LL`, `^MD` 등)은 첫 필드보다 **앞에** 와야 한다.
- 본 제품은 펌웨어 x.14 이상 Link-OS 계열 프린터를 기준으로 한다(현행 ZPL II 전부 지원).
- 참고 문서: Zebra "ZPL II, ZBI 2, Set-Get-Do, Mirror, WML Programming Guide"
  (P1134473, 2026-04 개정판 기준).

### 3.3 좌표계와 단위
- 원점 (0,0) = 라벨 좌상단. X축 →, Y축 ↓.
- 변환식: `dots = mm × dpi / 25.4`, `dots = inch × dpi`.
  - 예) 4인치 폭 @203dpi = 812 dot, @300dpi ≈ 1200 dot, @600dpi ≈ 2400 dot.
- 모든 디자인은 **DPI 비의존 내부 모델(기준 단위 mm)**로 저장하고, 내보내기 시
  대상 DPI로 dot 환산한다. DPI 변경 시 좌표는 `target_dpi / source_dpi` 비율로 재계산.
- `^FO`(Field Origin)는 요소의 **좌상단**을 기준으로 한다. 본 제품은 일관성을 위해
  `^FO`를 기본 채택한다(`^FT` baseline 방식은 내부적으로 사용하지 않음).

### 3.4 라벨 구조
```
^XA
^PW812            ; 인쇄 폭(dot)
^LL1218           ; 라벨 길이(dot)
^CI28             ; UTF-8 인코딩
^FO50,50^A0N,40,40^FDHello^FS
^FO50,120^BY3^BCN,80,Y,N,N^FD123456789^FS
^XZ
```

### 3.5 회전 제약 (중요)
ZPL은 **0°/90°/180°/270°** 네 각도만 지원한다(`N/R/I/B`). 자유 각도(예: 45°) 회전은
불가능하다. 따라서 본 제품의 회전 조작은 **90° 단위로 강제(snap)**하며, 그 외 각도는
허용하지 않는다.

### 3.6 흑백 단색 출력
ZPL 프린터는 흑백 1비트 출력만 가능하다. "색상"은 검정(`B`)·흰색(`W`)·반전(`R`)뿐이다.
이미지는 모두 1비트 비트맵으로 디더링(dithering)되어 변환된다.

## 4. 범위 (Scope)

### 4.1 In Scope
- WYSIWYG 캔버스 라벨 에디터 (단일 라벨/페이지).
- 디자인 요소: 텍스트, 1D 바코드, 2D 코드, 선, 박스, 원, 타원, 대각선, 이미지, 테이블, 심볼.
- ZPL II 코드 실시간 생성 / 복사 / 파일 다운로드.
- ZPL 코드 가져오기(역파싱) — v1.1.
- 로컬 저장(IndexedDB), 프로젝트 파일 `.json` 입출력, 내장 템플릿.
- 데이터 바인딩(변수 필드, CSV 머지) — 단계적.
- 경량 프린터 프로필(DPI · 라벨 크기 · 폰트셋).

### 4.2 Out of Scope (현 버전)
- 사용자 계정 / 클라우드 저장 / 실시간 협업 (백엔드 없음 결정에 따름).
- 프린터로의 직접 인쇄 전송(USB/네트워크).
- ZPL 외 프린터 언어(TSPL, EPL, DPL 등) 코드 생성.
- 프린터 모델별 상세 프로필(200+ Zebra 모델 개별 선택).

### 4.3 향후 검토 (Future)
- 코드 생성기를 플러그인화하여 TSPL/DPL 등 다국어 출력 지원.
- 프린터 모델 선택(모델 → DPI·폭·폰트 자동 설정).
- Labelary API 연동 실측 프리뷰(네트워크 정책 허용 시).
- PWA 오프라인 설치, 브라우저→프린터 전송 브리지.

## 5. 프린터 프로필 & 하드웨어 정책

> 조사 결론: ZPL 명령어 집합은 Zebra 모델 간 **동일**하다. 모델별로 달라지는 것은
> 주로 **DPI(좌표 스케일)**, **인쇄 폭**, **폰트 가용성**이다. 따라서 모델별 상세
> 선택은 불필요하며, 경량 프로필로 충분하다.

### 5.1 프린터 프로필 개념
프로필은 출력 정확도를 위한 최소 정보 묶음이다.
```
PrinterProfile {
  name: string                 // 예: "Generic 203dpi"
  language: "zpl"              // 향후 확장 대비 필드
  dpi: 203 | 300 | 600
  maxPrintWidthMm: number      // 물리 인쇄 폭 상한
  safeFonts: string[]          // 모든 모델 공통 폰트 (예: ["0","A","B"])
}
```
- 기본 제공 프리셋: **Generic 203dpi**(기본값), **Generic 300dpi**, **Generic 600dpi**.
- 사용자는 프리셋 선택 또는 DPI·폭 직접 입력. 프로필은 프로젝트에 저장된다.

### 5.2 DPI 처리
- 디자인은 DPI 비의존(mm 기준)으로 저장. 내보내기 시 대상 DPI로 환산.
- DPI 변경 시 전 요소 좌표·크기를 비율 재계산하고, 변경 전 미리보기를 제공한다.

### 5.3 폰트 가용성
- 안전 폰트셋(스케일러블 `0`, 비트맵 `A`~`H`)을 기본 노출.
- 비트맵 폰트는 DPI에 따라 두께가 달라지므로 스케일러블 폰트 `0`을 기본 권장.
- 한글 등 CJK는 내장 폰트로 출력 불가 → **§6.3.1 / §11** 참고(다운로드 폰트 안내·경고).

### 5.4 다른 프린터 언어 정책
| 언어 | 제조사 | 본 제품 지원 |
| --- | --- | --- |
| ZPL / ZPL II | Zebra | **v1 지원** |
| TSPL/TSPL2 | TSC | 향후(아키텍처상 확장 가능) |
| DPL | Datamax-O'Neil(Honeywell) | 향후 |
| EPL, CPCL, SBPL, IPL | Zebra(구형)/SATO/Honeywell | 미지원 |

코드 생성부는 `CodeGenerator` 인터페이스(Strategy 패턴)로 분리해, 디자인/캔버스
계층을 수정하지 않고 새 언어 생성기를 추가할 수 있도록 설계한다(**§9 참고**).

## 7. ZPL 매핑 명세

### 7.1 라벨/설정 매핑
| 디자인 항목 | ZPL 명령 | 비고 |
| --- | --- | --- |
| 라벨 시작/끝 | `^XA` … `^XZ` | 모든 출력의 외곽 |
| 인쇄 폭 | `^PWx` | x = 폭(dot) |
| 라벨 길이 | `^LLx` | x = 길이(dot) |
| 인코딩 | `^CI28` | UTF-8 |
| 농도 / 속도 / 수량 | `^MDx` / `^PRx` / `^PQx` | 출력 설정 |
| 미디어 추적 / 모드 | `^MN` / `^MM` | 연속·다이컷, 컷터 등 |
| 라벨 전체 회전 | `^POn` | n ∈ N/R/I/B |
| 주석 | `^FX … ` | 옵션 시 삽입 |

### 7.2 요소 매핑
| 요소 | ZPL 명령 | 핵심 파라미터 |
| --- | --- | --- |
| 텍스트 | `^FO` + `^A`(또는 `^A@`) + `^FD` … `^FS` | 폰트ID, 높이·폭, 회전 N/R/I/B |
| 여러 줄 텍스트 | + `^FB` (구형) / `^TB` (Link-OS) | 블록 폭, 최대 줄, 줄 간격, 정렬 |
| 반전 텍스트/도형 | + `^FR` | 흰색/검정 반전 |
| 1D 바코드 | `^FO` + `^BY` + `^BC`/`^B3`/`^BE`/`^BU`/`^B2`… + `^FD` … `^FS` | 종류, 높이, HRT, 체크디지트 |
| 2D 코드 | `^FO` + `^BQ`/`^BX`/`^B7`/`^B0`/`^BD` + `^FD` … `^FS` | 배율, 오류정정, 모드 |
| 직선 | `^FO` + `^GB` (h=1 또는 w=1) … `^FS` | 두께(t), 색(B/W) |
| 박스/사각형 | `^FO` + `^GBw,h,t,c,r` … `^FS` | 폭·높이·테두리·색·라운딩 |
| 대각선 | `^FO` + `^GDw,h,t,c,o` … `^FS` | 폭·높이·두께·방향 |
| 원 | `^FO` + `^GCd,t,c` … `^FS` | 지름·두께·색 |
| 타원 | `^FO` + `^GEw,h,t,c` … `^FS` | 폭·높이·두께·색 |
| 이미지 | `^FO` + `^GFA,…` … `^FS` (또는 `~DG`+`^XG`) | 1비트 비트맵, 폭 8-dot 배수 |
| 테이블 | 다수 `^GB` + 셀별 `^FO`/`^A`/`^FD`/`^FS` | 선·셀 합성 전개 |
| 그래픽 심볼 | `^FO` + `^GS` … `^FS` | 심볼·크기 |

### 7.3 좌표 변환 규칙
- 내부 저장: mm 기준. 내보내기: `dot = round(mm × dpi / 25.4)`.
- 회전: 요소 회전값을 `N(0) / R(90) / I(180) / B(270)`로 매핑. 그 외 각도는 입력 단계에서 차단.
- 이미지: 폭을 8-dot 배수로 올림 정렬 후 비트맵 생성.

| 2D 코드 렌더 | qrcode 등 | QR/2D 캔버스 미리보기용 |
| 이미지 디더링 | 자체 구현(Floyd-Steinberg) | 1비트 변환 |
| 파일 저장 | file-saver | `.zpl`/`.json` 다운로드 |

> 캔버스 라이브러리 조사 결과 Konva가 본 용도에 가장 적합. 단, 텍스트 인캔버스 편집은
> HTML 오버레이(textarea) 패턴이 필요하며 이는 라벨 디자인 툴 업계 표준 방식이다.
> ZPL 생성은 정밀 제어를 위해 자체 모듈로 구현하되, 오픈소스(JSZPL, zpl-js)를 참고한다.

### 9.2 핵심 계층 구조
```
┌─────────────────────────────────────────┐
│  UI 계층 (React + Konva 캔버스, 패널)       │
├─────────────────────────────────────────┤
│  상태 계층 (Zustand: 문서·선택·히스토리)     │
├─────────────────────────────────────────┤
│  도메인 모델 (Project / Label / Element)   │
├─────────────────────────────────────────┤
│  CodeGenerator 인터페이스                  │
│    └ ZplCodeGenerator (v1)               │
│    └ (향후) TsplCodeGenerator …           │
├─────────────────────────────────────────┤
│  영속화 (IndexedDB) · 파일 입출력           │
└─────────────────────────────────────────┘
```
- **CodeGenerator 인터페이스**: `generate(label: Label, options): string`.
  디자인 모델과 코드 생성을 분리 → 향후 다른 프린터 언어 확장 시 디자인/캔버스 계층 무수정.
- **ZPL 파서**(v1.1): 가져오기용. `parse(zpl: string): Label`.

### 9.3 폴더 구조(제안)
```
src/
  components/      # 캔버스, 패널, 툴바 등 UI
  canvas/          # Konva 래퍼, 변환 핸들, 스냅, 눈금자
  elements/        # 요소 타입별 모델·렌더러
  state/           # Zustand 스토어, undo/redo
  codegen/         # CodeGenerator 인터페이스 + ZplCodeGenerator
  parser/          # ZPL 파서 (v1.1)
  persistence/     # IndexedDB(Dexie), 파일 입출력
  printer/         # 프린터 프로필·DPI 변환
  templates/       # 내장 템플릿
  utils/           # 단위 변환, 디더링 등
```

interface Project {

### 10.2 요소(Element) 공통 + 타입별
```ts
interface ElementBase {
  id: string;
  type: 'text' | 'barcode1d' | 'barcode2d' | 'line' | 'box'
      | 'circle' | 'ellipse' | 'diagonal' | 'image' | 'table' | 'symbol';
  xMm: number; yMm: number;       // 좌상단 기준
  widthMm: number; heightMm: number;
  rotation: 0 | 90 | 180 | 270;
  locked: boolean;
  visible: boolean;
  zIndex: number;
  groupId?: string;
}

interface TextElement extends ElementBase {
  type: 'text';
  content: string;                // {{변수}} 포함 가능
  fontId: string;                 // '0','A'~'H'
  fontHeightDot: number; fontWidthDot: number;
  align: 'L' | 'C' | 'R';
  multiline?: { blockWidthMm: number; maxLines: number; lineGapDot: number };
  reverse?: boolean;              // ^FR
}

interface Barcode1DElement extends ElementBase {
  type: 'barcode1d';
  symbology: 'CODE128' | 'CODE39' | 'CODE93' | 'EAN13' | 'EAN8'
           | 'UPCA' | 'UPCE' | 'ITF' | 'CODABAR' | 'MSI';
  data: string;
  moduleWidthDot: number;         // ^BY x
  ratio: number;                  // ^BY y
  showHrt: boolean;
  checkDigit: boolean;
}

interface Barcode2DElement extends ElementBase {
  type: 'barcode2d';
  symbology: 'QR' | 'DATAMATRIX' | 'PDF417' | 'AZTEC' | 'MAXICODE';
  data: string;
  magnification: number;
  errorCorrection?: 'L' | 'M' | 'Q' | 'H';
}

interface ShapeElement extends ElementBase {       // line/box/circle/ellipse/diagonal
  borderThicknessDot: number;
  color: 'B' | 'W';
  filled: boolean;
  cornerRadiusDot?: number;       // box 전용
}

interface ImageElement extends ElementBase {
  type: 'image';
  sourceDataUrl: string;
  dither: 'floyd-steinberg' | 'threshold' | 'none';
  threshold: number;
  invert: boolean;
}

interface TableElement extends ElementBase {
  type: 'table';
  rows: number[]; cols: number[]; // 각 행 높이·열 폭(mm)
  cells: { row: number; col: number; rowSpan: number; colSpan: number;
           text: string; align: 'L' | 'C' | 'R'; fontId: string }[];
  borderThicknessDot: number;
  showInnerBorders: boolean;
}
```

## 14. 리스크 & 미해결 질문

### 14.1 리스크
| 리스크 | 영향 | 대응 |
| --- | --- | --- |
| ZPL 자유 각도 회전 미지원 | 디자인 자유도 제약 | 회전 90° 스냅 강제 + UI 안내 |
| 한글/CJK 내장 폰트 부재 | 한글 라벨 출력 실패 | 다운로드 폰트 안내·경고, 화면 미리보기는 대체 폰트 |
| 이미지→ZPL 용량 과다 | 전송 지연·메모리 초과 | Z64 압축, 크기·용량 경고, `~DG` 재사용 권장 |
| 프린터 펌웨어별 차이(`^TB`·라운딩 등) | 일부 명령 미동작 | 호환성 경고, 구형 호환 명령(`^FB`) 우선 |
| 캔버스 미리보기 ≠ 실제 출력 | 사용자 혼동 | (v1.1) Labelary 실측 프리뷰 옵션 제공 |

### 14.2 미해결 질문
- 한 프로젝트에서 여러 라벨(멀티 페이지)을 다룰 것인가, 1라벨 1프로젝트로 둘 것인가?
- Labelary 실측 프리뷰는 외부 API 의존 — 네트워크 정책상 허용 여부 확인 필요.
- 프린터 직접 전송(WebUSB/네트워크 브리지)을 어느 시점에 도입할지.

| --- | --- |
