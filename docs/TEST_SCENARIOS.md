# 테스트 시나리오 & 테스트 환경 가이드

ZPL Designer를 검증하기 위한 테스트 환경과 시나리오 문서.

## 1. 테스트 환경 구성

### 1.1 자동화 테스트
| 종류 | 도구 | 실행 명령 | 위치 |
| --- | --- | --- | --- |
| 단위 테스트 | Vitest | `npm test` | `tests/unit/` |
| E2E 테스트 | Playwright | `npm run test:e2e` | `tests/e2e/` |
| 타입 체크 | tsc | `npm run typecheck` | 전체 |
| 프로덕션 빌드 | Vite | `npm run build` | 전체 |

E2E를 처음 실행하기 전 브라우저 설치:
```bash
npm run test:e2e:install   # playwright chromium 설치
npm run test:e2e
```

### 1.2 Chrome DevTools MCP (대화형 브라우저 테스트)
레포 루트의 `.mcp.json`에 **공식 `chrome-devtools-mcp`**가 등록되어 있다.
Claude Code가 이 MCP로 실제 브라우저를 띄워 앱을 조작·검사할 수 있다.

```jsonc
// .mcp.json
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["-y", "chrome-devtools-mcp@latest", "--headless", "--isolated"]
    }
  }
}
```

사용 절차:
1. 개발 서버 실행: `npm run dev` (기본 `http://localhost:5173`)
2. Claude Code 세션을 새로 시작하면 `.mcp.json`의 MCP가 로드된다.
3. `chrome-devtools` MCP 도구(`navigate_page`, `take_snapshot`, `click`,
   `fill`, `take_screenshot`, `list_console_messages` 등)로 아래 시나리오를 수행한다.
4. 로컬에서 화면을 직접 보려면 `.mcp.json`의 `--headless`를 제거한다.

> 참고: 샌드박스 네트워크 정책에 따라 브라우저 바이너리 다운로드가 차단될 수 있다.
> 그 경우 로컬 환경(브라우저 설치 가능)에서 실행한다.

## 2. 수동/대화형 테스트 시나리오

각 시나리오는 `npm run dev` 실행 후 `http://localhost:5173`에서 수행한다.
ZPL 출력은 하단 패널(`data-testid="zpl-output"`)에서 확인한다.

### S1. 앱 로드
1. 페이지 접속.
2. **기대**: 상단에 "ZPL Designer", 중앙 흰색 라벨 캔버스, 하단 ZPL 코드 패널에
   `^XA … ^XZ`가 표시된다. 콘솔 오류 없음.

### S2. 텍스트 추가 및 편집
1. 좌측 도구 팔레트에서 **텍스트** 클릭.
2. 캔버스에 텍스트 요소가 생기고 우측 속성 패널이 열린다.
3. 속성 패널에서 내용을 "배송 라벨", 폰트 높이를 60으로 변경.
4. **기대**: 캔버스와 ZPL(`^A0N,60,…^FD배송 라벨^FS`)이 즉시 갱신된다.

### S3. 1D 바코드 / 2D 코드
1. **바코드** 추가 → 심볼 `EAN13`, 데이터 `123` 입력.
2. **기대**: 속성 패널에 빨간 경고("EAN-13은 숫자 12자리…"), 상태바에 오류 배지.
3. 데이터를 `123456789012`로 수정 → 경고 사라짐, ZPL에 `^BE` 출력.
4. **QR/2D** 추가 → 데이터·오류정정(H) 설정 → ZPL에 `^BQ`, `^FDHA,…` 확인.

### S4. 도형 (선·박스·원·타원·대각선)
1. 박스·선·원·타원·대각선을 각각 추가.
2. 박스: 채우기 토글, 라운딩 0→6 변경 → 펌웨어 의존 경고 확인.
3. **기대**: ZPL에 `^GB`, `^GC`, `^GE`, `^GD`가 각각 출력된다.

### S5. 이미지
1. **이미지** 추가 → 속성 패널에서 PNG/JPG 파일 업로드.
2. 디더링(Floyd-Steinberg ↔ 임계값), 임계값 슬라이더, 반전 조절.
3. **기대**: 캔버스에 1비트 변환 미리보기, ZPL에 `^GFA,…` 출력. 큰 이미지면 용량 경고.

### S6. 테이블
1. **테이블** 추가 → 행 3, 열 3.
2. 속성 패널 셀 그리드에 텍스트 입력.
3. **기대**: 캔버스에 표가 그려지고 ZPL에 다수의 `^GB` + 셀별 `^FO/^FD` 출력.

### S7. 선택 · 이동 · 크기 · 회전
1. 요소 클릭 선택 → 드래그 이동(그리드 스냅), 변환 핸들로 크기 조절.
2. 회전 핸들을 돌리면 0/90/180/270°로 스냅된다.
3. Shift+클릭/마키 드래그로 다중 선택.
4. **기대**: 속성 패널 수치와 ZPL 좌표가 일치한다.

### S8. 정렬 · 분배 · 그룹 · z-순서
1. 요소 3개 이상 선택 → 정렬/가로·세로 균등 분배.
2. 그룹(Ctrl+G) → 그룹 이동 → 해제(Ctrl+Shift+G).
3. 레이어 패널에서 ▲▼로 순서 변경, 표시/잠금 토글.

### S9. 실행취소 / 재실행
1. 요소 추가 후 Ctrl+Z → 제거, Ctrl+Y → 복원.
2. 복사(Ctrl+C)/붙여넣기(Ctrl+V)/복제(Ctrl+D)/삭제(Del) 확인.

### S10. 템플릿
1. 툴바 **열기 / 템플릿** → "배송 라벨" 선택.
2. **기대**: 변수 `{{받는사람}}` 등이 포함된 디자인이 로드된다.

### S11. 데이터 바인딩 & CSV 머지
1. 우측 **데이터** 탭 → 변수 추가, 샘플 값 입력.
2. "캔버스에 샘플 값 표시" 토글 → `{{변수}}`가 샘플 값으로 렌더.
3. CSV 업로드 → 레코드 ◀▶ 이동 → "이 레코드를 샘플 값에 적용".

### S12. ZPL 내보내기 & 다국어 생성기
1. 하단 패널에서 주석/UTF-8/변수치환 옵션 토글.
2. **복사**·**다운로드** 동작 확인.
3. 생성기를 **TSPL**로 전환 → `SIZE … CLS … PRINT` 출력(플러그인 아키텍처).

### S13. ZPL 가져오기 (역파싱)
1. 툴바 **ZPL 가져오기** → 샘플 ZPL 붙여넣기 → 가져오기.
2. **기대**: 명령이 캔버스 요소로 복원된다.

### S14. 저장 / 불러오기
1. 디자인 변경 후 자동저장(0.8초) 또는 **저장** 클릭.
2. 새로고침 → 마지막 프로젝트가 복원된다(IndexedDB).
3. **JSON↓/JSON↑**로 프로젝트 파일 입출력.

### S15. 인쇄 미리보기
1. 툴바 **미리보기** → Labelary 실측 렌더 이미지 표시(네트워크 필요).
2. CSV 로드 시 레코드 이동 + 전체 ZPL 다운로드.

### S16. 설정
1. **설정** → 테마(라이트/다크), 단위(mm/inch/dot), 그리드/스냅 변경.
2. **기대**: 캔버스·눈금자·속성 패널에 즉시 반영된다.
